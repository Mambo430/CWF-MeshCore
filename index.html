What is MeshCore?

Node Roles & Radio Settings

Roles

  • Companion: Intended role for end‚Äëuser devices that provide an interface via the phone or web app (via BLE or USB).
  • Repeater: Nodes in this role extend the range of a MeshCore network. They do not forward or retransmit every packet they receive.
  • Room Server: Turns the node into a simple BBS server for sharing posts. Room servers store message history on them and push the stored messages to users, allowing roaming users to come back later and retrieve history.
  • Room‚ÄëRepeater: A combination of Room Server and Repeater roles. Install the Room Server, then enable the repeater role via the set repeat {on|off} command.

Member Projects

The MeshCore ecosystem is growing thanks to a handful of community‚Äëbuilt tools. Below is a quick snapshot of the most popular open‚Äësource projects you can fork, use or contribute to.

  • MeshCore SAME EAS Alerter

  • meshcore‚Äëha

    A custom Home Assistant integration that lets you **monitor and control** MeshCore mesh‚Äëradio nodes directly from HA. It supports USB, BLE, and TCP connections, so you can bring your entire network into the same UI you already use for lights, sensors, etc.

  • meshcore‚Äëcli

    CLI tool to interact with a companion radio node (MeshCore client) over BLE, TCP or serial. Supports both interactive mode and scripted commands. Note: only companion radios can be reached (repeater serial interfaces are not supported).

  • meshcore_py

    Python library for interacting with MeshCore companion radio nodes.

  • meshcore‚Äëdecoder

    TypeScript library that decodes MeshCore mesh packets with full cryptographic support, using WebAssembly (WASM) for Ed25519 key derivation.

  • meshcore‚Äëmqtt‚Äëbroker

    WebSocket‚Äëbased MQTT broker that authenticates clients via MeshCore public keys.

  • meshcoretomqtt

    Python script that forwards MeshCore debug / packet‚Äëcapture data to an MQTT broker. Requires a MeshCore repeater to be connected to a Raspberry‚ÄØPi (or similar) that runs Python.

  • meshcore‚Äëpacket‚Äëcapture

    Stand‚Äëalone Python script for capturing and analysing packets from MeshCore Companion radios only. Connects via BLE, serial or TCP, writes structured data to console, file and optionally an MQTT broker.

MQTT Client Configuration

The following settings are used to connect to the public Florida MQTT broker that feeds the Florida Live Map. If you want to be added to the Live Map, add these environment variables to the bottom of your .env.local. You can take a look at the .env.local.example file. 

# MQTT Broker 3 - Florida MQTT Broker (US)
MCTOMQTT_MQTT3_ENABLED=true
MCTOMQTT_MQTT3_SERVER=mcmqtt.jntconnections.com
MCTOMQTT_MQTT3_PORT=443
MCTOMQTT_MQTT3_TRANSPORT=websockets
MCTOMQTT_MQTT3_USE_TLS=true
MCTOMQTT_MQTT3_USE_AUTH_TOKEN=true
MCTOMQTT_MQTT3_TOKEN_AUDIENCE=mcmqtt.jntconnections.com

Firmware

MeshCore has four firmware types that are not available on other LoRa systems. (The detailed list is on the Firmware Flasher page.)

Companion Radio Firmware

Companion radios are for connecting to the Android app or web app as a messenger client. Two companion radio firmware versions:

  • BLE Companion ‚Äì runs on a supported LoRa device and connects to a smart device running the Android or iOS MeshCore client over BLE. https://meshcore.co.uk/apps.html
  • USB Serial Companion ‚Äì runs on a supported LoRa device and connects to a smart device or computer over USB‚ÄØSerial running the MeshCore web client. https://meshcore.liamcottle.net/#/ https://client.meshcore.co.uk/tabs/devices

Repeater

Repeaters are used to extend the range of a MeshCore network. Repeater firmware runs on the same devices that run client firmware. A repeater forwards MeshCore packets only to the intended destination, not every packet it receives (unlike many other LoRa mesh systems).

A repeater can be remotely administered with a Lilygo T‚ÄëDeck running MeshCore firmware with remote‚Äëadmin features unlocked, or from a BLE Companion client connected to a smartphone running the MeshCore app.

Room Server

A room server is a simple BBS‚Äëstyle server for sharing posts. T‚ÄëDeck devices running MeshCore firmware or a BLE Companion client connected to a smartphone running the MeshCore app can connect to a room server.

Room servers store message history on the server and push stored messages to users. This lets roaming users come back later and retrieve message history‚Äîmuch like an email server.

When a client logs into a room server, the client will receive the last 32 unseen messages. While a room server can be set to ‚Äúrepeat on‚Äù, that mode is not recommended because it lacks the full set of repeater and remote‚Äëadmin features that only exist in the repeater firmware.

The recommendation is to run repeater and room server on separate devices for the best experience.
